### 功能概述

注册页是用户创建账户的入口页面。用户填写企业账户名、管理员名称、密码、手机号/邮箱及验证码等信息，勾选协议后提交注册。注册成功后自动跳转至首页。

### 交互要点

- 表单包含：企业账户名、管理员名称、密码、确认密码、手机号/邮箱、验证码、协议勾选
- 企业账户名：3-10位，不能含空格，汉字/字母/数字/下划线/减号组合，只能以汉字或字母开头
- 管理员名称：非空，3-10位，汉字
- 密码：6-16位，包含数字和字母，支持明文/密文切换
- 确认密码：6-16位，包含数字和字母，与密码一致，支持明文/密文切换
- 手机号/邮箱：手机格式校验 / 邮箱格式校验
- 验证码：非空，点击发送按钮前先校验手机号/邮箱格式，发送后 60 秒倒计时
- 协议勾选：必须勾选后才能提交
- 仅在提交时进行表单校验，校验失败用 Toast 提示第一个错误
- 邀请码从路由参数中获取（如从分享链接跳转带参）
- 注册成功后使用 `router.replace()` 跳转到首页

### T3：Auth Store（认证状态管理）

**触发 Skill**：`create-store`

**需求描述**：

创建认证状态管理 Store，用于存储用户登录/注册后返回的认证信息（token 和用户信息）。token 会被请求拦截器读取并注入请求头，用户信息会在多个页面中使用。Store 需要开启持久化，确保刷新/重启后认证状态不丢失。同时提供清除认证信息的能力（用于登出场景）。

**使用的已有资源**：
- 无

**预期产出**：
- [ ] `src/store/auth.ts`（新建 Auth Store）
- [ ] `src/store/index.ts`（更新桶文件导出）

### T5：验证码倒计时 Hook

**触发 Skill**：`create-hook`

**需求描述**：

创建管理验证码发送和倒计时的 Hook。功能包括：
1. 触发发送验证码（需要根据输入内容判断是手机号还是邮箱，调用对应的验证码发送 API）
2. 发送成功后启动 60 秒倒计时
3. 倒计时期间禁止重复发送
4. 提供当前倒计时剩余秒数和是否正在倒计时的状态
5. 组件卸载时清理倒计时定时器

**使用的已有资源**：
- `didappSendPhoneAuthCode` API（发送手机验证码）
- `didappSendEmailAuthCode` API（发送邮箱验证码）

**预期产出**：
- [ ] `src/hooks/use-auth-code.ts`（新建 Hook）
- [ ] `src/hooks/index.ts`（更新桶文件导出）

### T6：注册页业务逻辑

**触发 Skill**：`create-logic`

**需求描述**：

创建注册页的完整业务逻辑（无设计稿，template 和 style 仅做基本占位）。页面逻辑包括：

1. 从路由参数中获取邀请码
2. 管理表单数据（企业账户名、管理员名称、密码、确认密码、手机号/邮箱、验证码）
3. 管理协议勾选状态
4. 密码和确认密码的明文/密文切换
5. 发送验证码前先校验手机号/邮箱格式，通过后调用发送
6. 点击注册按钮时：先检查协议勾选 → 表单校验（仅提交时校验，Toast 提示第一个错误） → 调用注册 API
7. 确认密码需与密码一致的校验
8. 注册成功后：将返回的 token 和用户信息存入 Auth Store → 使用 `router.replace()` 跳转到首页
9. 注册按钮的 loading 状态管理

**使用的已有资源**：
- `didappCreateUser` API（注册接口）
- `useFormValidation` Hook（表单校验框架）
- `useLoading` Hook（加载态管理）
- `usePasswordVisibility` Hook（T4 产出）
- `useAuthCode` Hook（T5 产出）
- `useAuthStore` Store（T3 产出）
- 校验工具函数（T2 产出）
- `router` 工具（路由跳转）
- `PAGE_HOME` 常量（首页路径）
- `PAGE_REGISTER` 常量（T1 产出）

**预期产出**：
- [ ] `src/pages/register/index.vue`（注册页面）
