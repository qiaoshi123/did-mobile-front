## 一、需求摘要

### 页面基本信息

| 项目 | 内容 |
|------|------|
| 页面名称 | 注册页 |
| 页面路由 | `/pages/register/index` |
| 路径常量名 | `PAGE_REGISTER` |
| 页面类型 | 表单页 |
| 主包/分包 | 主包 |
| 导航栏标题 | 注册 |
| 导航栏字体颜色 | 继承全局 |
| 导航栏背景色 | 继承全局 |
| 导航栏模式 | 系统默认 |
| tabBar 页面 | 否 |

### 功能概述

注册页是用户创建企业账户的入口。用户需填写企业账户名、手机号/邮箱、密码、确认密码、管理员名称、验证码等信息，勾选协议后完成注册，注册成功后自动登录并跳转首页。

### 交互要点

- 表单校验在点击"下一步"按钮时统一触发，不做实时校验
- 校验失败时以 Toast 提示第一条不符合规范的提示语
- 手机号/邮箱使用单一输入框，前端智能判断输入的是手机号还是邮箱
- 密码和确认密码均支持小眼睛切换明文/密文
- 协议未勾选时以 Toast 提示"请先同意协议"
- 发送验证码后进入 60 秒倒计时，倒计时期间按钮禁用
- 发送验证码时根据输入内容自动判断调用手机号或邮箱的验证码接口
- 邀请码（invitationCode）从路由 query 参数中获取，不在表单中展示
- 注册成功后使用 `router.replace()` 跳转首页，用户无法返回注册页
- 注册成功后将返回的 token 和用户信息存入全局状态

### T2：新建 Auth Store

**触发 Skill**：`create-store`

**需求描述**：

新建认证状态管理模块，用于存储登录/注册后返回的 token 和用户信息。token 被请求拦截器跨模块使用，用户信息被多个页面展示使用，均需全局共享。需开启持久化存储，确保刷新或冷启动后状态不丢失。

需要存储的数据：
- token（登录凭证）
- 用户信息（对应 API 返回的用户信息结构）

需要提供的能力：
- 设置/清除 token
- 设置/清除用户信息
- 判断是否已登录

**使用的已有资源**：
- 无

**预期产出**：
- [ ] `src/store/auth.ts`（新建）
- [ ] `src/store/index.ts`（修改，导出新 Store）

### T3：新建校验工具函数

**触发 Skill**：`create-util`

**需求描述**：

创建一组表单校验工具函数，用于注册页（及未来可能的登录页等）的表单字段校验。每个校验函数接收待校验的值，返回校验是否通过的结果。

需要创建的校验函数：
- 企业账户名校验：3-10 位，不能含空格，汉字/字母/数字/下划线/减号组合，只能以汉字或字母开头
- 手机号格式校验：标准手机号正则校验
- 邮箱格式校验：标准邮箱正则校验
- 密码校验：6-16 位，必须包含数字和字母
- 管理员名称校验：非空，3-10 位，纯汉字
- 判断输入是手机号还是邮箱的工具函数（供发送验证码时判断调用哪个接口）

**使用的已有资源**：
- 无

**预期产出**：
- [ ] `src/utils/validate.ts`（新建）
- [ ] `src/utils/index.ts`（修改，导出新模块）

### T6：注册页业务逻辑

**触发 Skill**：`create-logic`

**需求描述**：

实现注册页的完整业务逻辑，包括表单管理、校验流程、注册提交和页面跳转。无设计稿，template 和 style 仅做最基础占位，重点实现逻辑层。

表单字段：
- 企业账户名
- 手机号/邮箱（单一输入框）
- 密码
- 确认密码
- 管理员名称
- 验证码
- 协议勾选状态

核心业务流程（点击"下一步"按钮）：
1. 检查协议是否勾选，未勾选则 Toast 提示"请先同意协议"并终止
2. 执行表单校验（统一校验所有字段），校验失败则 Toast 提示第一条不符合规范的提示语并终止
3. 确认密码需额外校验与密码是否一致
4. 从路由 query 参数获取邀请码
5. 调用注册接口
6. 注册成功后将返回的 token 和用户信息存入 Auth Store
7. 使用 `router.replace()` 跳转首页

其他交互逻辑：
- 密码和确认密码各自使用密码切换 Hook 控制明文/密文
- 验证码区域使用发送验证码 Hook（含倒计时）
- 注册请求使用加载态管理

**使用的已有资源**：
- API `didappCreateUser`（已有）
- Hook `useFormValidation`（已有，校验框架）
- Hook `useLoading`（已有，加载态）
- Hook `usePasswordToggle`（T4 产出）
- Hook `useSendCode`（T5 产出）
- Store `auth`（T2 产出）
- Util 校验工具函数（T3 产出）
- Util `router`（已有，路由跳转）

**预期产出**：
- [ ] `src/pages/register/index.vue`（新建）
