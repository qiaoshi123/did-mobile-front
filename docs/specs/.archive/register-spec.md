### 功能概述

注册页是用户创建企业账户的入口页面。用户填写企业账户名、手机号/邮箱、密码、确认密码、管理员名称、验证码，勾选协议后完成注册。注册成功后存储登录凭证和用户信息，跳转替换到首页。

### 交互要点

- 手机号/邮箱为同一个输入框，自动识别用户输入的是手机号还是邮箱格式
- 密码和确认密码输入框各带一个眼睛图标，可切换明文/密文显示
- 点击"发送验证码"前先校验手机号/邮箱格式，格式不正确时不允许发送
- 发送验证码后启动 60 秒倒计时，倒计时期间按钮不可点击
- 表单校验仅在点击"下一步"按钮时统一触发
- 校验失败时以 Toast 方式提示第一条不符合规范的错误信息，不在输入框旁 inline 展示错误状态和提示文字
- 协议未勾选时以 Toast 方式提示用户
- 注册失败时以 Toast 方式展示错误信息
- 注册成功后使用 `router.replace()` 跳转到首页（替换当前页，不保留注册页在页面栈中）
- 邀请码从路由 query 参数中获取，不在表单中展示

### T6：注册页 SFC

**触发 Skill**：`create-sfc`

**需求描述**：

创建注册页完整 SFC 文件，包含以下功能模块：

1. **表单区域**：包含企业账户名、手机号/邮箱、密码、确认密码、管理员名称、验证码共 6 个输入框
2. **密码输入框交互**：密码和确认密码输入框各带一个眼睛图标，点击切换明文/密文显示
3. **验证码发送**：手机号/邮箱输入框旁有"发送验证码"按钮，点击前先校验手机号/邮箱格式（校验函数返回错误信息字符串或空字符串），格式不正确时 Toast 提示错误信息且不发送，通过后发送验证码并启动倒计时
4. **手机号/邮箱自动识别**：同一个输入框，根据输入内容自动判断是手机号还是邮箱，调用对应的验证码发送 API
5. **协议勾选**：表单底部的协议勾选区域（checkbox + 文字）
6. **下一步按钮**：点击后先检查协议勾选（未勾选则 Toast 提示），再进行表单统一校验（校验失败时以 Toast 方式提示第一条不符合规范的错误信息，不在输入框旁展示错误状态），校验通过后对 password 字段进行 base64 编码，然后调用注册 API
7. **邀请码处理**：页面加载时从路由 query 参数中获取邀请码，作为注册 API 的参数传递
8. **注册成功处理**：调用 Auth Store 存储 token 和用户信息，然后使用 `router.replace()` 跳转到首页
9. **注册失败处理**：以 Toast 方式展示错误信息
10. **确认密码校验**：验证确认密码与密码值是否一致

**使用的已有资源**：
- `didappCreateUser` API（注册接口）
- `useFormValidation` Hook（表单校验框架）
- `useLoading` Hook（加载态管理）
- `usePasswordToggle` Hook（T3 产出，密码切换）
- `useSendCode` Hook（T4 产出，验证码发送）
- Auth Store（T5 产出，认证状态存储）
- 校验工具函数（T2 产出，各字段格式校验）
- `router` 工具（路由跳转）
- TDesign 组件：`t-input`、`t-button`、`t-checkbox` 等

**关键交互逻辑**：
1. 页面加载时从路由 query 参数中获取邀请码并缓存
2. 用户在手机号/邮箱输入框输入内容，系统自动识别格式（手机号或邮箱）
3. 用户点击"发送验证码"→ 先校验手机号/邮箱格式 → 格式正确则根据识别结果调用对应 API 发送验证码 → 发送成功后启动 60 秒倒计时
4. 用户点击密码输入框的眼睛图标 → 切换该输入框的明文/密文显示（两个密码框独立控制）
5. 用户点击"下一步"→ 检查协议是否勾选（未勾选 Toast 提示并中止）→ 表单统一校验（校验失败以 Toast 提示第一条错误信息，不 inline 展示错误）→ 对 password 字段进行 base64 编码 → 调用注册 API
6. 注册成功 → Auth Store 存储 token 和用户信息 → `router.replace()` 到首页
7. 注册失败 → Toast 展示错误信息

**预期产出**：
- [ ] `src/pages/register/index.vue`（新建）
