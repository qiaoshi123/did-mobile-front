# Spec 归档目录

## 目录用途

此目录用于存放已完成的任务归档，保留历史记录供开发人员查看。

## 目录结构

```
.archive/
└── <page-name>/
    └── <version>-tasks.md
```

## 示例

```
.archive/
├── login/
│   ├── v1.0-tasks.md    # 登录页第一版任务归档
│   └── v1.1-tasks.md    # 登录页第二版任务归档
└── register/
    └── v1.0-tasks.md    # 注册页第一版任务归档
```

## 归档文件内容

每个归档文件包含：
- 归档时间
- 原始版本号
- 已完成任务列表
- 任务详情（保留原始描述）
- 历史决策记录

## 归档触发机制

1. **手动归档**：用户主动要求归档已完成任务
2. **自动归档**：活跃文档中已完成任务超过 5 个时自动触发

## 归档流程

1. 扫描活跃文档 (`docs/specs/<name>-spec.md`)，找出所有 ✅ 已完成任务
2. 生成归档文件到 `docs/specs/.archive/<name>/<version>-tasks.md`
3. 从活跃文档中删除已完成任务
4. 更新活跃文档的"变更记录"章节

## 注意事项

- 归档文档仅供历史追溯，不影响新任务的生成
- 每次生成提案都是全新分析代码库，以代码为真相源
- 归档文档不会被删除，永久保留
