## 一、需求摘要

### 功能概述

优化注册页的表单校验方法：将校验函数从返回布尔值改为直接返回错误信息，错误提示文案从页面逻辑层下沉到 util 工具函数层，使校验函数自包含错误信息，页面层无需再硬编码校验提示语。

### 改造要点

- 校验函数校验通过时返回 `true`，不通过时返回对应的错误信息字符串（与现有 `useFormValidation` 的 validator 返回值约定一致）
- 不保留原有返回布尔值的校验函数，直接改造为返回错误信息的形式
- `getContactType` 等内部调用校验函数的工具函数需要适配新的返回值
- 页面中 `useFormValidation` 的 rules 配置简化，校验逻辑直接调用工具函数即可，不再在页面中编写 validator 包装函数和硬编码错误提示

## 三、任务详情

### T1：改造校验工具函数返回值

**触发 Skill**：`create-util`

**需求描述**：

修改 `src/utils/validate.ts` 中的所有表单字段校验函数（企业账户名、手机号、邮箱、密码、管理员名称），将返回值从布尔值改为：校验通过返回 `true`，校验不通过返回对应的中文错误提示信息字符串。

每个校验函数需自包含错误提示文案，具体错误信息参照当前页面中已有的提示语：
- 企业账户名：格式不符时提示账户名的格式要求
- 手机号：格式不符时提示手机号格式错误
- 邮箱：格式不符时提示邮箱格式错误
- 密码：格式不符时提示密码的格式要求
- 管理员名称：格式不符时提示管理员名称的格式要求

同时 `getContactType` 函数内部调用了手机号和邮箱校验函数，需适配新的返回值（`getContactType` 自身返回值类型不变）。

更新 `src/utils/index.ts` 中的导出（如有变化）。

**使用的已有资源**：
- `src/utils/validate.ts`（现有校验函数，直接改造）
- `src/utils/index.ts`（统一导出）

**预期产出**：
- [ ] `src/utils/validate.ts`（修改）
- [ ] `src/utils/index.ts`（如导出有变化则修改）

### T2：适配注册页业务逻辑

**触发 Skill**：`create-logic`

**需求描述**：

修改 `src/pages/register/index.vue` 中的 `useFormValidation` rules 配置，移除所有硬编码的错误提示信息和 validator 包装函数。因为校验函数已自包含错误信息，页面中的自定义 validator 可以直接调用校验工具函数作为 validator，无需再包装一层。

涉及的表单字段：企业账户名、手机号/邮箱、密码、确认密码、管理员名称。

注意：确认密码字段除了密码格式校验外，还有"两次密码一致性"校验逻辑，这部分属于页面特有的业务逻辑，仍保留在页面中。

**使用的已有资源**：
- `src/utils/validate.ts`（T1 改造后的校验函数）
- `src/hooks/use-form-validation.ts`（已有表单校验 Hook）

**预期产出**：
- [ ] `src/pages/register/index.vue`（修改）
