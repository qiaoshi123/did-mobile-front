# 组件规范

## 组件分类与放置

| 分类 | 位置 | 适用场景 |
|------|------|---------|
| 通用组件 | `src/components/<name>/` | 被 2 个及以上页面复用的 UI 组件 |
| 页面级组件 | `src/pages/<page>/components/<name>/` | 仅当前页面使用的组件 |

- `src/components/` 只存放真正通用的组件，禁止放入仅单页使用或业务强耦合的组件
- 页面级组件被第二个页面引用时，必须提升到 `src/components/`

## 组件目录结构

每个组件独占一个 kebab-case 目录，内部文件固定：

| 文件 | 必须 | 说明 |
|------|------|------|
| `index.vue` | ✅ | 组件主文件（SFC） |
| `types.ts` | 当 Props/Emits 较复杂时 | 独立类型定义（简单 Props 可内联在 SFC 中） |

## 组件统一导出

- 所有 `src/components/` 下的通用组件必须在 `src/components/index.ts` 中统一导出
- 每条导出必须附带 JSDoc 注释，说明组件的用途和适用场景
- 页面级组件不需要在全局 `index.ts` 中导出

## 组件选择优先级

编写页面时，组件选择遵循以下优先级（高 → 低）：

| 优先级 | 来源 | 信息获取方式 | 说明 |
|--------|------|------------|------|
| 1 | 本地通用组件 | 读取 `src/components/index.ts` | 已有的直接用，通过 JSDoc 注释了解用途 |
| 2 | TDesign 组件 | 查阅 `tdesign-uniapp` Skill 中的组件文档 | 确认 Props/Events/Slots 后再使用 |
| 3 | 新建组件 | — | 以上都不满足时才新建 |

- 禁止跳过优先级 1、2 直接新建组件
- 使用本地组件前，必须读取 `src/components/index.ts` 确认可用组件
- 使用 TDesign 组件前，必须通过 `tdesign-uniapp` Skill 确认准确用法，禁止凭记忆猜测

## TDesign 组件使用约束

- TDesign 组件必须按需引入，禁止全量引入
- 使用任何 TDesign 组件前，必须通过 `tdesign-uniapp` Skill 确认：组件标签名、Props、Events、Slots
- 二次封装 TDesign 组件时，封装组件放在 `src/components/` 下，组件内部引用 TDesign 原始组件
- 当 TDesign 组件不满足需求需要二次封装时，封装后的组件须在 `src/components/index.ts` 中导出

## Props 与 Emits 契约

- Props 和 Emits 使用 TypeScript 泛型语法定义（`defineProps<T>()`、`defineEmits<T>()`）
- Props 接口中每个字段必须附带 JSDoc 注释
- 可选 Props 必须通过 `withDefaults()` 设置默认值
- 禁止直接修改 Props，通过 `emit` 通知父组件

## 组件通信方式

| 场景 | 方式 | 说明 |
|------|------|------|
| 父 → 子 | Props | 单向数据流 |
| 子 → 父 | Emits | 事件通知 |
| 双向绑定 | `v-model` / `update:xxx` | 语法糖 |
| 祖孙跨层级 | `provide` / `inject` | 需声明类型 |
| 全局状态 | Pinia Store | 跨组件共享状态 |

## 命名约定

| 对象 | 规则 | 示例 |
|------|------|------|
| 组件目录名 | kebab-case | `did-card`、`security-modal` |
| 组件导出名 | PascalCase | `DidCard`、`SecurityModal` |
| Props 接口名 | `<ComponentName>Props` | `DidCardProps` |
| Emits 接口名 | `<ComponentName>Emits` | `DidCardEmits` |
| 事件命名 | kebab-case（template 中） | `@item-click`、`@update:visible` |

## 组件复杂度约束

- 单个 `.vue` 文件建议不超过 400 行（超出时考虑拆分子组件）
- 遵循单一职责原则，每个组件应有明确的职责边界
- 组件 template 中禁止写复杂逻辑，提取为 `computed` 或方法

## 禁止事项

| 序号 | 禁止行为 | 说明 |
|------|---------|------|
| 1 | 禁止直接修改 Props | 通过 `emit` 通知父组件 |
| 2 | 禁止在组件内直接使用 uni API 进行路由跳转 | 路由跳转由页面层处理，组件通过 `emit` 通知 |
| 3 | 禁止使用内联样式（动态样式除外） | 使用 `<style scoped>` 或 class 绑定 |
| 4 | 禁止全量引入 TDesign | 必须按需引入具体组件 |
| 5 | 禁止通用组件不在 `index.ts` 中导出 | 所有通用组件必须统一注册导出 |
| 6 | 禁止导出时缺少 JSDoc 注释 | 组件导出必须附带用途说明 |
