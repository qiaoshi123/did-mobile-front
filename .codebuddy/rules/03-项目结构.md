# 项目结构

## 目录结构

```
src/
├── config/        # 应用配置（网关地址、环境变量）
├── http/          # HTTP 请求模块（core + interceptors + services 三层架构）
├── pages/         # 页面目录（uni-app 路由页面）
├── components/    # 通用组件（跨页面复用）
├── store/         # Pinia Store，按业务拆分
├── utils/         # 通用工具函数
├── constants/     # 业务常量、枚举
├── types/         # 全局类型定义（非 API 相关）
├── static/        # 静态资源（图片、图标）
└── 根级文件        # App.vue、main.ts、pages.json、manifest.json、uni.scss 等
```

## 结构约束（禁止在 src 下新建非标准目录）

| 类型 | 目录 | 规范 |
|------|------|------|
| 配置 | `src/config/` | `gateway.json` 为网关唯一数据源，`index.ts` 统一导出 |
| HTTP | `src/http/` | 三层架构（core + interceptors + services） |
| 页面 | `src/pages/` | 每个页面一个目录，目录名 kebab-case，主文件固定 `index.vue` |
| 组件 | `src/components/` | 可复用的UI组件 |
| 状态 | `src/store/` | 按业务模块划分，暴露 `useXxxStore` Hook，统一导出到 `index.ts` |
| 工具 | `src/utils/` | 按功能模块划分，统一导出到 `index.ts` |
| 常量 | `src/constants/` | UPPER_SNAKE_CASE 命名，统一导出到 `index.ts` |
| 类型 | `src/types/` | 非 API 相关的全局业务类型（API 类型放在 `http/services/` 下） |
| 静态资源 | `src/static/` | 图片放 `images/`，图标放 `icons/`，文件名 kebab-case |

## 页面目录内的组件放置规则

- 仅当前页面使用的组件放在 `src/pages/<page-name>/components/` 目录下
- 被 2 个及以上页面复用的组件应提升到 `src/components/`
- `src/components/` 只存放通用组件，禁止将仅单页使用的组件放入

## 页面路由约束

- 每个页面必须在 `src/pages.json` 中注册路由
- 路由 `path` 不以 `/` 开头、不含 `.vue` 后缀
- `pages` 数组第一项为应用启动页

## 目录创建原则

- 新建目录前先确认是否已有合适的目录可用
- 按需创建，不要提前创建空目录
- 避免超过 3 层嵌套
- 模块内聚，同一功能的文件放在同一目录，通过 `index.ts` 统一导出
