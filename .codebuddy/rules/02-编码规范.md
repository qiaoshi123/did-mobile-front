# 编码规范

## TypeScript 规范

- 所有代码必须使用 TypeScript，禁止使用 JavaScript
- 禁止使用 `any` 类型（不确定时使用 `unknown`）
- 项目开启了 `verbatimModuleSyntax`，必须区分值导入和类型导入：类型用 `import type`，值正常导入，禁止混用
- 纯类型文件导出使用 `export type`
- 接口定义优先使用 `interface`，联合类型/交叉类型使用 `type`

## Vue 3 组件规范

- 所有组件和页面统一使用 `<script setup lang="ts">` 语法
- SFC 结构顺序：`<template>` → `<script setup lang="ts">` → `<style lang="scss" scoped>`
- Props 和 Emits 使用泛型语法定义（`defineProps<{...}>()`、`defineEmits<{...}>()`）
- 禁止直接修改 prop，通过 `emit` 通知父组件
- 禁止在 template 中写复杂逻辑，提取为 computed 或方法
- 页面中使用 uni-app 的生命周期钩子（`onLoad`、`onShow` 等，从 `@dcloudio/uni-app` 导入）

## 命名规范

| 类型 | 命名规则 | 示例 |
|------|----------|------|
| 文件夹/页面目录 | kebab-case | `user-detail`、`login` |
| 变量/函数 | camelCase | `userList`、`onSubmit` |
| 常量 | UPPER_SNAKE_CASE | `API_TIMEOUT` |
| 接口/类型 | PascalCase | `UserInfo`、`DidappLoginBody` |
| Vue 组件文件 | `index.vue`（目录名为组件名） | `user-card/index.vue` |
| CSS 类名 | kebab-case | `.user-card` |
| 自定义 Hooks | use 开头 | `useAuth` |

## 业务函数命名

- **事件处理函数**：`onXxx`（如 `onBannerClick`、`onFormSubmit`）
- **内部处理函数**：`handleXxx`（如 `handleDelete`、`handleSubmit`）

## 格式化规则

- 缩进：4 空格
- 字符串：单引号（模板字符串用反引号）
- 语句末尾：不加分号
- 对象/数组：最后一项加尾逗号
- 每行最大长度：推荐不超过 120 字符

## 禁止事项

| 序号 | 禁止行为 | 说明 |
|------|---------|------|
| 1 | 禁止使用 `any` | 使用 `unknown` 或明确类型替代 |
| 2 | 禁止使用 `// @ts-ignore` | 应修复类型错误，而非忽略 |
| 3 | 禁止使用 `require` | 统一使用 `import` |
| 4 | 禁止直接修改 prop | 通过 `emit` 通知父组件修改 |
| 5 | 禁止在 template 中写复杂逻辑 | 提取为 computed 或方法 |
| 6 | 禁止类型和值的 import 混用 | `verbatimModuleSyntax` 要求分开写 |
