# 第 2 步：区域与元素提取

## 概述

对**每个区域**按「从外到里」逐项提取。**文字、图片、布局、层级四者均为重中之重，必须逐一准确记录。**

详见 `analysis-priorities.md`。

---

## 2.1 提取顺序

对布局 Map 中的每个区域，按以下顺序提取：

1. **层级结构**（从外到里的树形图）
2. **文字信息**（每处可见文案）
3. **图片/图标信息**（每处图片、图标、占位图）
4. **布局参数**（尺寸、间距、对齐）
5. **样式细节**（背景色、边框、圆角、阴影）

---

## 2.2 层级结构（必须先做）

先画出该区域的树形嵌套结构，确定父子关系，再逐个填充细节。

**格式示例**：

```
导航栏容器（row, space-between）
├── 左侧：返回图标 24×24rpx
├── 中间：标题文字 "登录"
└── 右侧：占位（无元素，保持居中）
```

```
列表项容器（row）
├── 左侧图标 40×40rpx
├── 中间内容区（column, flex:1）
│   ├── 标题文字 "安全设置"
│   └── 描述文字 "修改密码、手机号"
└── 右侧箭头图标 16×16rpx
```

### 常见层级模式（移动端）

#### 卡片类

```
卡片容器（圆角 + 背景色 + padding）
├── 标题区（row）
│   ├── 标题文字
│   └── 更多/操作按钮
├── 内容区
│   └── 具体内容...
└── 底部区（可选）
    └── 操作按钮/说明文字
```

#### 列表类

```
列表容器
├── 列表项 1（row）
│   ├── 左侧元素
│   ├── 中间内容
│   └── 右侧元素
├── 分隔线
├── 列表项 2
│   └── ...
└── 列表项 N
```

#### 表单类

```
表单容器
├── 表单项 1（row）
│   ├── 标签文字 "手机号"
│   └── 输入框（placeholder: "请输入手机号"）
├── 表单项 2
│   └── ...
└── 提交按钮
```

## 2.3 文字信息

每处可见文案记录：

| 字段 | 必须 | 说明 |
|------|------|------|
| 内容 | ✅ | 完整文案或典型示例 |
| 字号 | ✅ | rpx（基于 750 基准）|
| 字重 | ✅ | 400（regular）/ 500（medium）/ 600（semibold） |
| 颜色 | ✅ | hex 值 + 对应 TDesign 变量（如有） |
| 行高 | 可选 | 如能从截图判断 |
| 最大行数 | 可选 | 是否有截断（如"最多2行，溢出省略"） |

**示例**：

```markdown
#### 文字
| 元素 | 内容 | 字号 | 字重 | 颜色 | 备注 |
|------|------|------|------|------|------|
| 页面标题 | "登录" | 34rpx | 600 | #333 → var(--td-text-color-primary) | |
| 手机号标签 | "手机号" | 28rpx | 400 | #666 → var(--td-text-color-secondary) | |
| 输入框占位 | "请输入手机号" | 28rpx | 400 | #ccc → var(--td-text-color-placeholder) | placeholder |
| 提交按钮 | "登录" | 32rpx | 500 | #fff | 按钮内文字 |
```

---

## 2.4 图片/图标信息

| 字段 | 必须 | 说明 |
|------|------|------|
| 类型 | ✅ | icon / image / placeholder |
| 尺寸 | ✅ | 宽 × 高（rpx） |
| 位置 | ✅ | 在哪个区域/元素内 |
| 圆角 | 可选 | rpx |
| 说明 | ✅ | 什么内容/用途 |

**示例**：

```markdown
#### 图片/图标
| 元素 | 类型 | 尺寸(rpx) | 圆角 | 说明 |
|------|------|----------|------|------|
| 返回箭头 | icon | 48×48 | - | 导航栏左侧返回按钮 |
| 用户头像 | image | 120×120 | 60rpx(圆形) | 个人中心头像 |
| 右箭头 | icon | 32×32 | - | 列表项右侧指示箭头 |
```

---

## 2.5 布局参数

记录每个区域/元素的布局信息：

| 字段 | 说明 |
|------|------|
| 排列方向 | row（横向）/ column（纵向） |
| 宽度 | rpx 或 满宽 / flex:1 / 固定值 |
| 高度 | rpx 或 自适应 |
| 内边距 | padding（rpx，标注方向） |
| 元素间距 | gap（rpx） |
| 对齐方式 | 左 / 居中 / 右 / 两端对齐 + 垂直方向的对齐 |

**示例**：

```markdown
#### 布局
- 排列方向：row
- 宽度：满宽 750rpx
- 高度：88rpx
- 内边距：padding: 0 32rpx
- 元素间距：gap: 16rpx
- 水平对齐：space-between
- 垂直对齐：center
```

## 2.7 常见遗漏检查（每个区域提取完后过一遍）

- [ ] 输入框的 placeholder 文案

- [ ] 分隔线
- [ ] 底部安全区域
- [ ] 空状态（如列表为空时的提示）
