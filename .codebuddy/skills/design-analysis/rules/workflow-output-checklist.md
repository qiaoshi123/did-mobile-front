# 第 5 步：输出文档要求

## 概述

将分析结果输出为「UI 清单」文档。本文档说明文档路径、结构和质量要求。

---

## 文档路径

```
docs/ui-analysis/<页面或模块名称>-ui-inventory.md
```

**命名规则**：
- 使用 kebab-case
- 以 `-ui-inventory` 结尾

**示例**：
- `docs/ui-analysis/login-ui-inventory.md`
- `docs/ui-analysis/user-center-ui-inventory.md`
- `docs/ui-analysis/product-detail-ui-inventory.md`

---

## 文档结构（必须包含）

详见 `output-ui-inventory.md` 获取完整模板。文档必须包含以下章节：

1. **文档说明** — 截图来源、分析日期、换算比例
2. **一、页面布局 Map** — 页面类型、区域列表、区域间关系
3. **二、区域详细分析** — 按从上到下逐区域，每区包含层级、文字、图片/图标、布局、样式
4. **三、样式规范汇总** — 颜色、字体、圆角、间距与 TDesign 变量的映射
5. **四、组件映射表** — 每个 UI 区块对应的组件方案
6. **五、交互状态说明** — 按钮状态、输入框状态等（如能从截图判断）
7. **六、遗漏检查记录** — 本次分析已核对的检查项
8. **七、给 create-proposal 的摘要** — 提炼出页面清单、组件清单、关键交互要点，便于 create-proposal 快速消费

---

## 文档质量要求

| 要求 | 说明 |
|------|------|
| **分析顺序** | 区域按「从上到下」书写，与前端布局一致 |
| **四类信息完整** | 文字、图片、布局、层级均不遗漏 |
| **尺寸已换算** | 所有尺寸值为 rpx（750 基准），不可留截图原始像素值 |
| **TDesign 映射** | 颜色/间距/圆角已与 `--td-xxx` 变量做映射 |
| **组件方案明确** | 每个 UI 区块都有对应的组件方案，标注来源和分类 |
| **可消费** | `create-proposal` 仅凭此文档 + API 文档 + 交互文档即可制定完整的实施计划 |

---

## 质量检查清单

输出文档前，检查以下项：

- [ ] 文档路径正确，符合命名规范
- [ ] 文档结构完整，包含全部 7 个章节
- [ ] 所有尺寸值已换算为 rpx
- [ ] 所有颜色/间距/圆角已与 TDesign CSS Variables 做映射
- [ ] 组件映射表覆盖所有 UI 区块
- [ ] 「给 create-proposal 的摘要」章节信息充分
- [ ] 分析顺序正确（从上到下、从左到右、从外到里）
