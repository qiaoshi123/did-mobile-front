# 第 4 步：组件映射

## 概述

将分析出的 UI 区块映射到具体的组件方案。这是本项目 design-analysis Skill **区别于通用设计稿分析**的关键步骤——不只是记录 UI 长什么样，还要明确**用什么组件来实现**。

组件映射的结果将直接作为 `create-proposal` Skill 的输入。

---

## 组件选择优先级（必守）

遵循 `04-组件规范` Rule 的优先级：

| 优先级 | 来源 | 说明 |
|--------|------|------|
| 1 | 本地通用组件（`src/components/`） | 已有的直接用 |
| 2 | TDesign 组件（`@tdesign/uniapp`） | 确认存在后再使用 |
| 3 | 新建组件 | 以上都不满足时才新建 |

**禁止跳过优先级 1、2 直接建议新建组件。**

---

## 映射流程

### 4.1 扫描已有组件

**必须先执行**：读取 `src/components/index.ts`，获取已有的通用组件清单。

记录格式：
```markdown
### 已有通用组件
| 组件名 | 用途 |
|--------|------|
| （当前暂无通用组件） | |
```

### 4.2 逐区域映射

对第 2 步提取出的每个 UI 区块，判断：

1. **是否匹配已有通用组件？** → 直接用，标注组件名
2. **是否匹配 TDesign 组件？** → 标注 TDesign 组件标签名（如 `<t-button>`、`<t-cell>`、`<t-input>`）
3. **都不匹配？** → 标注"需新建"，并确定分类：
   - 会被 ≥2 个页面复用 → **通用组件**（`src/components/`）
   - 仅当前页面使用 → **页面级组件**（`src/pages/<page>/components/`）

### 4.3 常见的 TDesign 移动端组件映射

> 以下为常见映射参考，**使用前必须通过 tdesign-uniapp Skill 确认组件是否存在及 Props**。

| UI 元素 | TDesign 组件 | 说明 |
|---------|-------------|------|
| 按钮 | `<t-button>` | 主按钮、次按钮、文字按钮 |
| 输入框 | `<t-input>` | 文本输入 |
| 单元格列表项 | `<t-cell>` | 左标题+右内容+箭头 |
| 弹出层 | `<t-popup>` | 底部弹出、居中弹出 |
| 对话框 | `<t-dialog>` | 确认/取消对话框 |
| 消息提示 | `<t-toast>` / `<t-message>` | 轻提示 |
| 加载 | `<t-loading>` | 加载中状态 |
| 图标 | `<t-icon>` | TDesign 图标库 |
| 标签 | `<t-tag>` | 标签/徽章 |
| 开关 | `<t-switch>` | 开关切换 |
| 导航栏 | `<t-navbar>` | 顶部导航 |
| TabBar | `<t-tab-bar>` | 底部导航 |
| 搜索框 | `<t-search>` | 搜索输入 |
| 下拉刷新 | `<t-pull-down-refresh>` | 列表下拉刷新 |
| 图片 | `<t-image>` | 图片展示（含加载中/失败状态） |
| 头像 | `<t-avatar>` | 头像展示 |
| 分隔线 | `<t-divider>` | 分隔线 |
| 空状态 | `<t-empty>` | 空状态页面 |

## 输出格式：组件映射表

```markdown
### 组件映射表

| 序号 | UI 区块 | 组件方案 | 来源 | 分类 | 备注 |
|------|---------|---------|------|------|------|
| 1 | 导航栏 | `<t-navbar>` | TDesign | - | 带返回按钮+标题 |
| 2 | 手机号输入 | `<t-input>` | TDesign | - | type="number" |
| 3 | 验证码输入+发送按钮 | 需新建 `verify-code-input` | 新建 | 通用组件 | 输入框+倒计时按钮组合 |
| 4 | 登录按钮 | `<t-button>` | TDesign | - | theme="primary" block |
| 5 | 协议勾选 | 需新建 `agreement-check` | 新建 | 页面级组件 | 勾选框+协议文字链接 |
```

### 字段说明

- **UI 区块**：第 2 步提取出的 UI 元素/区域名称
- **组件方案**：具体使用什么组件（TDesign 标签名 / 已有组件名 / 需新建的组件名）
- **来源**：已有组件 / TDesign / 新建
- **分类**：仅"新建"时需填写——通用组件 / 页面级组件
- **备注**：关键 Props、特殊说明

- [ ] 新建组件的分类正确（通用 vs 页面级）
