# 第 1 步：建立布局 Map

## 概述

建立布局 Map 是设计稿分析的第一步。目的是从截图中识别页面的整体结构，确定区域划分和区域间关系。

## 1.1 识别截图信息

### 确认截图尺寸

1. 查看截图的像素尺寸（宽 × 高）
2. 确认换算比例：`比例 = 750 ÷ 截图宽度`
3. 如果用户告知了设计稿实际宽度（如 750），直接使用
4. 如果未知，根据截图像素宽度推算或询问用户

### 识别页面类型

从截图中判断页面类型：
- **普通页面**：有导航栏 + 内容区 + 可能有底部操作栏
- **Tab 页面**：底部有 TabBar
- **弹窗/浮层**：半屏或全屏的浮层
- **列表页**：以列表为主体的页面
- **表单页**：以表单输入为主体的页面

## 1.2 多状态识别（如有多张截图）

如果用户提供了同一页面的**多张截图**（表示不同状态），执行以下步骤：

### 判断截图关系

1. **同一页面的不同状态**（如表单空态 vs 填写态、列表有数据 vs 空列表）→ 产出**一份** UI 清单
2. **不同页面**（如登录页 + 注册页）→ 分别产出各自的 UI 清单

### 选择基准态

从多张截图中选择一个作为**基准态**，按以下优先级：

| 优先级 | 选择标准 | 说明 |
|--------|---------|------|
| 1 | 信息最完整的状态 | 如表单**填写态**（能看到输入内容 + 按钮激活态） |
| 2 | 元素最多的状态 | 能展示更多 UI 元素的截图 |
| 3 | 默认/初始状态 | 如果各状态信息量差不多，选默认态 |

### 记录格式

```markdown
### 多状态说明
- **截图数量**：N 张
- **基准态**：截图 X（___态），理由：___
- **差异态列表**：
  - 截图 Y：___态
  - 截图 Z：___态
```

> **后续步骤（第 2~5 步）只对基准态做完整分析**，差异态在「第五章：多状态对比」中记录增量变化。

## 1.3 按「从上到下」扫描区域

**扫描顺序**：从截图顶部向下，逐一识别大的横向区域。

> **⚠️ 跳过系统状态栏**：移动端设计稿截图顶部通常带有手机系统状态栏（显示时间、电池电量、Wi-Fi 信号等），这部分是系统 UI，**不属于页面内容，必须跳过，不记录到区域列表中**。分析从状态栏下方的导航栏或页面内容区开始。

移动端页面常见的区域结构：

```
┌─────────────────────┐
│  系统状态栏（跳过！）  │ ← 不分析，不记录
├─────────────────────┤
│  导航栏              │ ← 分析从这里开始
├─────────────────────┤
│                     │
│  主内容区（可滚动）   │ ← 页面主体
│                     │
├─────────────────────┤
│  底部操作栏（如有）   │ ← fixed 底部按钮/TabBar
└─────────────────────┘
```

## 1.4 必须记录的内容

| 项目 | 说明 |
|------|------|
| 截图尺寸 | 截图像素尺寸 + 换算比例 |
| 页面类型 | 普通页/Tab 页/弹窗/列表页/表单页 |
| 背景色 | 页面整体背景色 |
| 区域列表 | **按从上到下排序**：区域名称、大致位置、包含的主要元素概述 |
| 区域间间距 | 相邻区域之间的垂直间距（rpx） |
| 固定区域 | 标注哪些区域是 fixed 定位（如导航栏、底部操作栏） |
| 滚动区域 | 标注哪些区域是可滚动的 |

## 1.5 布局 Map 模板

```markdown
## 页面布局 Map

### 截图信息
- 截图尺寸：___ × ___ px
- 换算比例：750 ÷ ___ = ___
- 页面类型：___
- 页面背景色：___

### 区域列表（从上到下）

| 序号 | 区域名称 | 定位方式 | 高度(rpx) | 主要元素概述 |
|------|---------|---------|----------|-------------|
| 1 | 导航栏 | fixed top | ≈88rpx | 返回按钮 + 页面标题 |
| 2 | 主内容区 | scroll | 自适应 | 表单项 × N + 说明文字 |
| 3 | 底部操作栏 | fixed bottom | ≈120rpx | 提交按钮 |

### 区域间间距
- 导航栏底部 → 主内容区顶部：___rpx
- 主内容区底部 → 底部操作栏顶部：___rpx（需预留底部栏高度）
```
