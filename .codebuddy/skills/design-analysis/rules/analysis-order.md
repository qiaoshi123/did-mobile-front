# 分析顺序（必守）

## 重要性

**必须按以下顺序分析设计稿截图，否则容易产生疏漏。**

不按顺序分析是分析遗漏与还原错误的主要来源之一。

## 分析顺序

### 1. 从上到下

先按垂直方向，从截图最上方扫到最下方，确定所有横向「带」与区块的先后顺序。

**为什么重要**：
- 与前端布局顺序一致（flex-direction: column），避免实现时顺序颠倒
- 确保顶部状态栏区域、导航栏不被遗漏
- 避免只关注主内容区而漏掉头部和底部

### 2. 从左到右

在同一行或同一层级内，再按从左到右依次记录，不跳过、不凭印象合并。

**为什么重要**：
- 避免只记录左侧内容而漏掉右侧（如列表项右侧的箭头图标）
- 确保同一层级内的元素顺序正确
- 避免凭印象合并相似元素，导致细节丢失

### 3. 从外到里

在每个区块内部，按「外层容器 → 内层容器 → 叶子元素」逐层进入，先轮廓后细节。

**为什么重要**：
- 确保层级嵌套关系正确，不丢层或顺序颠倒
- 先理解整体结构（如卡片容器），再深入细节（如卡片内的标题、描述、按钮）
- 便于后续映射到 Vue 组件的父子结构

## 扫描顺序示例（移动端页面）

```
从上到下：
  状态栏区域 → 导航栏 → 主内容区（可滚动） → 底部操作栏/TabBar

从左到右（以列表项为例）：
  头像 → 标题+描述（纵向） → 右侧箭头

从外到里（以卡片为例）：
  卡片容器 → 头部区域 → 头部内的标题文字 + 副标题文字
             内容区域 → 内容区的图片 + 描述
             底部区域 → 操作按钮
```

## 记录顺序

- 区域列表：按从上到下（y 坐标）排序
- 同层元素：从左到右（x 坐标）排序
- 嵌套结构：从外到里逐层进入
