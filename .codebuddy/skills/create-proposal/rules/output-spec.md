## 模板使用说明

1. **任务详情只写"需求描述"**：说清楚这个任务要实现什么功能，不要写字段定义、函数签名、代码示例等实现细节。具体实现由对应的 create-xxx Skill 根据需求和项目规范自行完成。
2. **决策记录是核心章节**：Spec 的长期价值在于决策记录。每个用户确认过的决策都要记录在"决策记录"章节。
3. **create-page 任务引用 UI 清单**：不复制样式数值，写明 UI 清单文件路径，让 create-page Skill 自行读取。
4. **资源方案总览合并为一章**：组件、Store、Hook、Util 的方案合并展示，一目了然。
5. **已有资源标注"直接使用"**：不为已有资源生成任务。
6. **未确认的信息标 ⚠️**：统一用"⚠️ 待补充"标注，不要自行填充。
