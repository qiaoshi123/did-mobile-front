## 一、需求摘要

### 页面基本信息

| 项目 | 内容 |
|------|------|
| 页面名称 | 注册页面 |
| 页面路由 | `/pages/register/index` |
| 路径常量名 | `PAGE_REGISTER` |
| 页面类型 | 表单页 |
| 主包/分包 | 主包 |
| 导航栏 | 系统默认 |
| tabBar 页面 | 否 |

### 功能概述

注册页面提供企业用户注册功能。用户填写企业账户名、手机号/邮箱、密码、管理员名称、验证码等信息，勾选协议后提交注册。邀请码从路由 query 参数中获取。注册成功后跳转到登录页面。

### 交互要点

- 企业账户名校验：3-10位，不能包含空格，汉字/字母/数字/下划线/减号或其组合，只能以汉字或字母开头
- 手机号/邮箱字段：需区分手机号和邮箱格式分别做正则校验
- 密码校验：6-16位，包含数字和字母
- 二次确认密码校验：6-16位，包含数字和字母，且与密码保持一致
- 管理员名称校验：非空，3-10位，汉字
- 验证码：非空
- 密码和确认密码字段均支持点击小眼睛切换明文/密文显示
- 协议勾选：单选，必须勾选才能提交
- 邀请码从路由 query 参数读取，不在表单中展示输入字段
- 发送验证码需区分手机号和邮箱，调用不同的 API
- 验证码发送后有倒计时效果
- 点击下一步按钮：检查协议勾选 → 表单验证 → 调用注册 API → 注册成功跳转登录页

## 三、任务详情

### T1：创建认证 Store

**触发 Skill**：`create-store`

**需求描述**：

创建认证 Store（auth），用于全局存储登录凭证（token）和用户信息。

- 存储登录凭证，需要被请求拦截器读取以注入请求头
- 存储用户信息（从 `didappCreateUser` API 返回的完整 useInfo 对象），包含用户ID、企业名、联系方式、邀请码、企业类型、成员角色、企业DID、企业公钥、个人DID、头像、管理员名称、VC失效状态等
- 需要提供设置凭证和用户信息的方法
- 需要提供清除认证状态（退出登录）的方法
- 需要提供判断是否已登录的 getter
- **开启持久化**（token 和用户信息需要在 app 重启后保留登录态）

**使用的已有资源**：

- `DidappAuthResult` 类型（`src/http/services/didapp-auth.ts` 中定义的认证返回类型）

**预期产出**：

- [ ] `src/store/auth.ts`
- [ ] 更新 `src/store/index.ts` 添加导出

### T5：创建注册页面逻辑

**触发 Skill**：`create-logic`

**需求描述**：

创建注册页面，包含完整的 script setup 逻辑和基础 template/style。

**表单字段**：
- 企业账户名
- 手机号/邮箱
- 密码（支持明文/密文切换）
- 确认密码（支持明文/密文切换）
- 管理员名称
- 验证码（附带发送验证码按钮）
- 协议勾选

**页面逻辑**：
1. 页面加载时从路由 query 参数中读取邀请码（invitationCode）
2. 使用 useFormValidation 组织表单校验：企业账户名、手机号/邮箱、密码、确认密码（需与密码一致）、管理员名称、验证码非空
3. 使用 usePasswordVisibility 分别管理密码和确认密码的明文/密文切换
4. 发送验证码时需判断当前输入是手机号还是邮箱，调用对应 API（手机号调用 didappSendPhoneAuthCode，邮箱调用 didappSendEmailAuthCode）
5. 发送验证码后通过 useCountdown 启动倒计时
6. 点击下一步按钮的流程：检查协议勾选 → 触发表单校验 → 调用 didappCreateUser API（传入企业账户名、手机号/邮箱、密码、管理员名称、验证码、邀请码） → 注册成功后将返回的 token 和用户信息存入 Auth Store → 跳转到登录页面
7. 使用 useLoading 管理注册请求的加载态

**路由配置**：
- 需在 pages.json 中新增注册页面路由配置
- 需在 src/constants/routes.ts 中新增 PAGE_REGISTER 和 PAGE_LOGIN 路径常量

**使用的已有资源**：

- `didappCreateUser` API（注册接口）
- `didappSendPhoneAuthCode` API（发送手机验证码）
- `didappSendEmailAuthCode` API（发送邮箱验证码）
- `useFormValidation` Hook（表单校验框架）
- `useLoading` Hook（加载态管理）
- `router` 工具函数（页面跳转）
- Auth Store（T1 产出）
- 校验工具函数（T2 产出）
- `useCountdown` Hook（T3 产出）
- `usePasswordVisibility` Hook（T4 产出）

**预期产出**：

- [ ] `src/pages/register/index.vue`
- [ ] 更新 `src/pages.json` 添加注册页面路由
- [ ] 更新 `src/constants/routes.ts` 添加 PAGE_REGISTER 和 PAGE_LOGIN 常量
