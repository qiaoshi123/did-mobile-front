# 编码规范

本文档定义 Vue 3 + TypeScript + UniApp 的编码规范。

## TypeScript 规范

### 基础类型

```typescript
// 正确：显式定义类型
const name: string = 'DID'
const count: number = 10
const isActive: boolean = true

// 错误：禁止使用 any
const data: any = getData()  // ❌
const data: unknown = getData()  // ✅
```

### 接口和类型

```typescript
// 用户信息
interface UserInfo {
  id: string
  name: string
  avatar?: string  // 可选属性
}

// 响应包装类型
interface ApiResponse<T> {
  code: number
  message: string
  data: T
}
```

### 泛型约束

```typescript
// 必须使用泛型约束，避免 any
function getData<T>(id: string): Promise<T> {
  return uni.request<T>({ url: `/api/${id}` })
}
```

## Vue 3 Composition API 规范

### setup 语法糖

```vue
<script setup lang="ts">
import { ref, computed, onMounted } from 'vue'
import type { UserInfo } from '@/types'

// 状态定义（必须标注类型）
const count = ref<number>(0)
const userInfo = ref<UserInfo | null>(null)

// 计算属性
const greeting = computed(() => `Hello, ${userInfo.value?.name || 'Guest'}`)

// 方法
function increment() {
  count.value++
}

// 生命周期
onMounted(() => {
  fetchUser()
})
</script>
```

### 组件通信

```vue
<!-- 父组件 -->
<ChildComponent
  :user-id="userId"
  @update="handleUpdate"
/>

<!-- 子组件 props -->
const props = defineProps<{
  userId: string
}>()

<!-- 子组件 emits -->
const emit = defineEmits<{
  (e: 'update', data: UserInfo): void
}>()
```

## 命名规范

| 类型 | 命名规则 | 示例 |
|-----|---------|------|
| 组件文件 | PascalCase | `UserCard.vue` |
| 组件名 | PascalCase | `UserCard` |
| 页面文件 | kebab-case | `user-detail.vue` |
| 函数/方法 | camelCase | `fetchUserInfo` |
| 变量/常量 | camelCase | `userList` |
| 常量 | UPPER_SNAKE_CASE | `API_TIMEOUT` |
| 类型/接口 | PascalCase | `UserInfo` |
| CSS 类 | kebab-case | `.user-card` |

## 导入规范

```typescript
// Vue 核心
import { ref, computed } from 'vue'
import type { Ref } from 'vue'

// 组件（按需导入）
import { Button, Input } from '@tdesign/uniapp'

// 类型
import type { UserInfo } from '@/types'

// 工具函数
import { formatDate } from '@/utils'

// 统一使用 @ 别名
// @ → src/
```

## 禁止事项

1. **禁止使用 `any`**
2. **禁止使用 `// @ts-ignore`**
3. **禁止使用 `require`（使用 import）**
4. **禁止直接修改 prop**
5. **禁止在 template 中使用箭头函数**

## 格式化规则

- 使用 2 空格缩进
- 语句结尾不加分号
- 对象最后一项末尾逗号
- 字符串使用单引号
